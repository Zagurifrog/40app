שלב 1 — עיצוב
הכל התחיל ב-Google Stitch

לפני שנכתב שורת קוד אחת, עברנו איטרציות עיצוב שלמות ב-Google Stitch. ניסינו גוונים, פונטים, פריסות כרטיסים, ואיך תראה חוויית 9:16 Story על מסך הנייד.

הגענו לשפה ויזואלית ברורה: רקע שחור עמוק, ניאון כתום, פונט Rubik כבד, RTL מלא בעברית. לא הייתה שאלה של "נראה טוב?" — הוא נראה טוב עוד לפני הקוד.

כשמגיעים לקוד עם עיצוב מוגמר — אין ניחושים. כל החלטה ויזואלית כבר התקבלה.
Dark cyberpunk פונט Rubik 900 RTL עברית פורמט 9:16 אפקט ניאון glow
🎨
שלב 2 — פיתוח
קובץ HTML אחד. אין שרת. אין build.

כל האפליקציה — HTML, CSS, JavaScript — נמצאת בקובץ index.html אחד. Tailwind דרך CDN, Google Fonts, ו-JS ישיר. אף פעמון, אף שרפרף.

תמונת טל עצמה הוטמעה כ-Base64 ישירות בתוך הקובץ — כדי שהיא תהיה שם תמיד, בכל מכשיר, גם בלי חיבור מוצלח לרשת. אורחים העלו תמונה שלהם דרך FileReader API, ומשם הכל זורם.

פרויקט מסיבה לא צריך Kubernetes. קובץ אחד ב-GitHub Pages = אפס נקודות כשל.
Tailwind CDN FileReader API Base64 embed GitHub Pages

שלב 3 — חיבור ל-AI
Gemini, אבל לא בלי מאבק

הרעיון: שולחים שתי תמונות (טל + האורח) ופרומפט של עשור ← מקבלים תמונה AI מוכנה לסטורי. בתיאוריה פשוט. בפועל — כל מודל שבחרנו הוחלף.

התחלנו עם gemini-2.0-flash-preview-image-generation, עברנו דרך gemini-2.0-flash-exp-image-generation, ונחתנו על gemini-2.5-flash-image — שנמצא ע"י שאילתה חיה ל-API כדי לראות מה בכלל קיים.

שמות מודלים משתנים בלי הודעה מוקדמת. תמיד תבדוק מה חי מול ה-API עצמו — לא מה כתוב בדוקומנטציה.

שלב 4 — מרתון הבאגים
שישה באגים. שישה שיעורים.
🔄 ה-placeholder הוחלף פעמיים
בדיקת הולידציה if (key === "PASTE...") הכילה גם היא את ה-placeholder — שניהם הוחלפו בערך האמיתי, אז הבדיקה תמיד ירתה.
🏷️ שם מודל שגוי
Google שינתה שמות בשקט. נאלצנו לשלוח בקשה חיה ל-API כדי לגלות את הרשימה האמיתית של המודלים הזמינים.
🔑 מפתח API דלף
הסורק הסודי של GitHub מצא את המפתח ב-repo הציבורי תוך דקות ובטל אותו. זה קרה פעמיים.
🐪 inlineData לא inline_data
ה-API מחזיר camelCase אבל הפרסר חיפש snake_case. שתיקה מושלמת — אין תמונה, אין שגיאה.
🚫 IMAGE_OTHER
Gemini סירב לייצר תמונות פוטוריאליסטיות של אנשים אמיתיים. הוספנו הודעת שגיאה ידידותית לאורחים.
📦 תמונה גדולה מדי
תמונת טל שוקלת 440KB — יותר מדי טוקנים. הוספנו resize ל-768px בצד הלקוח לפני השליחה.
🐛
שלב 5 — אבטחה
Cloudflare Worker: הפתרון האמיתי

אחרי שמפתח ה-API בוטל פעמיים, הגיע הזמן לפתרון אמיתי. הקמנו Cloudflare Worker — פונקציה serverless בחינם — שיושבת בין האפליקציה לבין Gemini.

המפתח חי כ-secret environment variable בתוך Cloudflare. הקוד הציבורי ב-GitHub לא מכיל מפתח בכלל. האפליקציה קוראת ל-gemini-proxy.zaguri.workers.dev — ה-worker מוסיף את המפתח ומעביר.

כל מפתח ב-repo ציבורי ייבדק ויבוטל תוך דקות. Worker proxy הוא הפתרון היחיד לאפליקציית client-side.
Cloudflare Workers (free) Secret env var CORS headers
